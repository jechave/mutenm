# Contributing to mutenm

Thank you for your interest in contributing to mutenm!

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub:
https://github.com/jechave/mutenm/issues

When reporting a bug, please include:
- A minimal reproducible example
- Your R version (`R.version.string`)
- Your operating system
- The output of `sessionInfo()`

## Contributing Code

1. Fork the repository
2. Create a new branch for your feature (`git checkout -b feature/my-feature`)
3. Make your changes
4. Run `devtools::check()` to ensure there are no errors or warnings
5. Submit a pull request

## Code Style

- Follow the tidyverse style guide
- Document all exported functions with roxygen2
- Add tests for new functionality

## Questions

For questions about using the package, please open an issue on GitHub.
